# idle-factory — 재사용 가능한 방치형 게임 템플릿

> 목표: 룡게임 모델의 재현 — 코드(엔진)는 재사용, 데이터(밸런스·컨텐츠)와 아트(스킨)만 교체해서
> 방치형 키우기 게임을 반복 출시한다. 벤치마크 분석: `../../research/soul-hunter-idle-analysis.md`

## 0. 설계 철학

1. **게임 = 코드 + 데이터 + 아트.** 양산이란 코드를 다시 안 짜는 것이다.
   신규 성장 축·가챠 배너·상품·이벤트는 **코드 수정 없이 JSON 추가**로 만들 수 있어야 한다.
2. **순수 C# 도메인.** 게임 로직은 MonoBehaviour 없는 `IdleCore`에 격리 → 헤드리스 테스트,
   밸런스 시뮬레이션, (미래의) 서버 검증 재사용이 가능하다. Unity는 뷰일 뿐이다.
3. **결정론.** 전투·오프라인 보상·가챠는 시드 주입 가능한 결정론적 계산 → 테스트와 시뮬레이터가
   실게임과 동일한 코드를 돌린다.
4. **오프라인 우선.** 로컬 저장이 진실이다. 서버(랭킹/쿠폰/클라우드 세이브)는 인터페이스만 두고
   나중에 뒤끝/Firebase 어댑터를 붙인다.

## 1. 레이어 구조

```
idle-factory/
├── Assets/
│   ├── IdleCore/            # 순수 C# 도메인 (Unity API 참조 금지, asmdef로 강제)
│   │   ├── Stats/           #   곱연산 스탯 시스템
│   │   ├── Economy/         #   재화, 상점, 패스, 페이백
│   │   ├── Progression/     #   스테이지, 전투 수식, 오프라인 보상
│   │   ├── Gacha/           #   배너, 천장, 돌파
│   │   ├── LiveOps/         #   이벤트 템플릿, 미션, 출석
│   │   ├── Save/            #   저장 모델 + 마이그레이션
│   │   └── Data/            #   테이블 로더/스키마
│   ├── IdleCore.Tests/      # EditMode 테스트 (NUnit)
│   ├── IdleGame/            # Unity 뷰 레이어 (UI, 씬, 이펙트)
│   └── GameData/            # JSON 테이블 (게임마다 교체되는 부분!)
├── tools/                   # Node 기반 밸런스 시뮬레이터·테이블 검증기
└── docs/
```

## 2. 핵심 시스템 명세 (소울헌터 플레이북 → 시스템 매핑)

### 2-1. StatSystem — 곱연산 성장 축
소울헌터의 파워커브 본질: `최종 데미지 = Π(축별 배율)`. "10×2×4 < 5×5×5".
- `GrowthAxis`(성장 축)는 **데이터로 등록**: `axes.json`에 축 id, 이름, 비용 곡선, 배율 곡선, 해금 조건.
- 격주 업데이트에서 신규 축(장비 증폭→재련→유물→…)을 추가하는 것 = JSON에 축 1개 추가.
- 축 내부는 가산, 축 사이는 곱산. `StatSnapshot`이 최종 수치를 캐시.

### 2-2. Economy — 재화·소탕권
- 이중 재화: `gem_hard`(유료, 원화 게이트) / `gem_soft`(보상). 소울헌터의 레드/블루 다이아.
- 컨텐츠별 재화·소탕권도 전부 `currencies.json` 데이터 정의. 소탕권이 병목 재화가 되도록
  던전 보상·쿠폰·패키지가 소탕권을 겨냥하는 구조를 데이터로 표현.

### 2-3. Progression — 스테이지·전투·오프라인
- 스테이지 = `챕터-단계`(예: 12-3). 보스 관문, 컨텐츠 해금을 스테이지에 결박(`unlocks.json`).
- 전투는 실시간 틱이 아니라 **수식 평가**: DPS vs 몬스터 EHP → 처치 시간 → 진행 속도.
  뷰는 결과를 연출만 한다 (방치형 표준).
- 오프라인 보상 = 경과시간 × 현재 스테이지 보상률 × 멤버십 배율, 상한 존재(멤버십으로 상한 확장
  → 소울헌터의 '별/달의 가호' 잠금 패턴).

### 2-4. Gacha — 배너·천장·돌파
- `banners.json`: 풀, 등급 확률, 픽업, 천장(150), X10/X50 할인.
- 중복 = 돌파 재료. **돌파 임계 설계**(예: 9돌파에서 핵심 효과 발동)를 데이터로 표현.
- 도감(컬렉션) 보유 효과: 모든 가챠 결과물이 계정 스펙에 기여 — '잉여 제로' 원칙.

### 2-5. Monetization — 가격 사다리·패스·페이백
- 상품(`products.json`): 하드커런시 가격 사다리 **490/990/1,990/2,990** 고정. 원화는 스토어에서만.
- 패스 3형: 진행도 패스(스테이지/던전 단계 연동, 상시) / 시즌 패스(월간 로테이션, 코스튬 트랙) /
  이벤트 패스(출석 990, 레이스 2,990).
- 결제 전환 퍼널 장치: **페이백 출석**(490 구매→7일 완주 시 전액 환급), **누적 사용 페이백**,
  **누적 구매 이벤트 상점**(hard 구매만 카운트), 1+1 재활성화, 진행도 트리거 패키지.
- `IStoreAdapter`(IAP) / `IAdAdapter`(보상형 광고) 인터페이스 + 개발용 FakeStore.
  실제 연동(Unity IAP, AppLovin MAX)은 어댑터 구현만 추가.

### 2-6. LiveOps — 이벤트 템플릿
소울헌터의 '6종 세트'를 하나의 이벤트 정의로 템플릿화(`events.json`):
출석(무료/유료 2트랙) + 미션(패스 시 2배) + 스텝업 + 교환소(+전용 코인) + 누적 사용 페이백 + 레이스.
스킨(이름/아트/보상 테이블)만 바꿔 격주 로테이션 — "새해맞이 → 썸머"의 재현.

### 2-7. Save — 로컬 우선
- JSON 직렬화 + `schemaVersion` 마이그레이션 체인. 치팅 방지는 1차 릴리즈 범위 아님(방치형 관례).
- `ICloudSave` 인터페이스만 정의 (뒤끝/Firebase는 어댑터).

### 2-8. 시간 규율
`IClock` 주입 — 오프라인 보상·출석·이벤트 기간 판정 전부 여기 경유. 테스트에서 시간 조작 가능.

## 3. 운영 케이던스 설계 (콘텐츠가 아니라 프로세스도 템플릿)

- **격주 정기 업데이트 = 신규 성장 축 1(JSON) + 가챠 유닛 1~2(JSON) + 보스/스테이지 확장(JSON)
  + 이벤트 스킨 교체(JSON)** — 전부 데이터 작업이 되도록 하는 것이 이 템플릿의 존재 이유.
- 빠른 출시·빠른 보정: 알려진 이슈 공지 + 보상 우편 템플릿을 운영 문서로 준비.

## 4. 1호 게임 테마 제안: 「저승사자 키우기: 방치형」

- 소울헌터/사신 계열의 **검증된 다크판타지 수요** + **한국 설화 차별화**(갓, 도포, 도깨비불, 삼도천).
- 서사와 BM의 자연 결합: 영혼 수확 = 소프트 재화, 명부(名簿) = 도감, 저승 문턱 = 스테이지,
  차사 동료·수문장 보스·달빛 각성 등 시스템 네이밍이 그대로 나옴.
- 아트: AI 생성으로 커버 가능한 실루엣 강한 2D 스타일. 경쟁작(사신 키우기)과 톤은 겹치나
  한국 저승 세계관 스킨은 비어 있음.
- 대안: 망나니 키우기(조선 처형인), 달빛 도깨비 키우기(라이트 톤).
